## Priority 1: Fix “Download Resume”

The button likely fails because the resume file is either:

1. missing from the public folder,
2. linked to the wrong path,
3. using a React route instead of a static asset path,
4. or the file name has spaces/capitalization that does not match.

Put the resume here:

```txt
/public/Kevin-Brandon-Corbett-Resume.pdf
```

Then the button should be:

```tsx
<a
  href="/Kevin-Brandon-Corbett-Resume.pdf"
  download="Kevin-Brandon-Corbett-Resume.pdf"
  className="resume-button"
>
  Download Resume
</a>
```

If it opens instead of downloading, that is still acceptable, but the `download` attribute should force download in most browsers.
